Linux-based Flow Meter - BeagleBone Socket/File MODBUS Integration

Tismo Technology Solutions - 2018 - 2020

Device communication uses sockets and file updates happen using BeagleBone. MODBUS register read operation reads from a file on the BeagleBone; write updates the device via socket communication and writes back the latest updates to the file from which MODBUS read happens.

  • MODBUS register update failed due to file corruption during socket communication (timeouts or packet drops when multiple commands were processed) — introduced mutexes and semaphores to fix it.
  • Multiple variable update issue when using multiple binaries was resolved by implementing atomic update strategies.
  • Crash fix: concurrent write and read on the same file caused crashes; implemented write-to-temp-and-rename, file-locking, and serialized access to prevent corruption.
BeagleBone Linux Sockets MODBUS Concurrency Mutex/Semaphore File I/O
